Java

Javass

The goal of this project, named Javass, is to write a program to play a variation of the famous Jass game. Jass is a very popular card game in Switzerland — where it is often referred to as a national sport — and other neighboring countries, and is closely related to belote and other similar games.

I have used the MVC pattern for the conception, the Monte Carlo Tree Search for the simulated player, Beans JavaFX for the Graphical Interface and used the EPFL VPN in order to allow people to play online.

Git Repository.

2D Game

Welcome to ENIGMA, a 2D grid based game. The player controls the protagonist from an overhead perspective and navigates him throughout a fictional region. The goal of the game is to solve all the riddles present in each world, there's a total of three worlds to visit. Enigma utilize a dynamic dialog, which makes the game more realistic for the player. it is necessary for players to escape from all the worlds in order to successfully complete the game.

Processing

Projet 1

This project consists of a tangible game on board. It will display a plate at the centre of the screen, with the sphere initially at the centre of it, the plate will tilt according to the movement of the board that the code reads from the video, and the sphere will roll on it naturally.

This project also includes some visualisation elements and shows the video augmented with detected lines and corners on a separate displaying window with image processing algorithms,like Hough transforms to find intersections and Scharr algorithm for edge detection.

Assembly

Tetris

Part of the CS208 course: Computer Architecture, this project is the implementation of the famous game TETRIS in assembly language. We were able to play it on Gecko4EPFL boards. The display of all the tetrominoes were used by bit manipulations. Each LED in the Gecko4EPFL board had a correspondent bit in a correspondent register, and the LED was off when its bit were at zero, and on when its bit is at one.

Git Repository.

C

Database

The project is an implementation of a database management program for storing the information of students participating in the course. The database is represented by a table of students, each one with a correspondent pair. The program also offers a set of tasks to manage the students and the pairs among them: the get least homogeneous teams filter,the get students by type function and other tasks.

Git Repository.

Todo List

This project consists of writing a small application that manages to-do lists (“TODO lists”) with various priorities.

In the first part, I implemented a queue ("queue" in English), which is a data structure based on the principle of first in, first out (FIFO, First In, First Out).

In the second part, I wrote a priority queue, which allows you to manage a task queue with different priorities. Your program thus allows you to add tasks (with their priority) to a priority queue, and return these tasks in the correct order of priority.

Git Repository.

VHDL

Multicycle Nios II Processor

In this program, I have implemented a multiyle Nios II processor that simulates the existence of a CPU unit, a Decoder a 4KB ROM, a 4KB RAM. The CPU’s controller have got different states, the most important states are the following

FETCH 1 : the CPU is set to start a new read process

FETCH 2: the instruction word is read and saved in a register.

R.OP: state executes operations between two registers

LOAD & STORE
